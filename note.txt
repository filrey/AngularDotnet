In order to recover from wonky migrations try this:
1: run dotnet ef database drop to remove current database
2: run 'dotnet ef migrations remove' to remove all migrations
3: run 'dotnet ef migrations add initialCommit'
4: run 'dotnet ef database update'

Running in Prod
1: NG build the SPA to wwwroot in DatingApp.API
2: Set "ASPNETCORE_ENVIRONMENT": "Production" in launchSettings.json
3: Set Azure Credentials in appsettings.json
4: use in cmd <set ASPNETCORE_ENVIRONMENT=Production> before running migrations
5: run dotnet publish -c release
6. deploy DatingApp.API\bin\release\netcoreapp3.0\publish to Azure
